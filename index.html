Sensor Introduction

Modern software projects are often built on top of third-party libraries. Dependency conflicts arise when the loaded versions of a project's libraries fail to cover or are incompatible with the required features referenced by the project. Dependency conflicts need to be carefully checked and resolved to avoid runtime exceptions or unexpected program behaviors. While existing compilers/linkers and static analyzers check the signature compatibility of the methods used in a project, they do not check the semantic compatibility of these methods and thus cannot catch semantic conflict issues caused by dependency conflicts. This motivates us to develop a technique that automatically detects semantic conflict issues, Sensor, which (1) performs static analysis to identify pairs of conflicting APIs; and (2) synthesizes test code with the injected valid arguments to invoke these API pairs and compares their test outcomes to identify real semantic conflict issues. Our evaluation results show that Sensor achieves a Precision of 0.803 and a Recall of 0.760 on open source projects and a Precision of 0.821 on industrial subjects. Sensor has identified 146 real semantic conflict issues from 28 projects. We merged these detected issues into 32 issue reports and submitted them to the developers of the corresponding projects. So far, 25 reported issues have been confirmed by developers as real bugs, and 19 have been quickly fixed. The results indicate the effectiveness and usefulness of our technique.
